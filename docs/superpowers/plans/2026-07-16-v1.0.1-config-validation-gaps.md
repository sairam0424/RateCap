# v1.0.1 Config Validation Gaps Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close 2 tracked follow-up issues (#10, #23) from RateCap's Tier 2/Tier 4 audits — both are "one bad or missing config value silently causes a 100% outage with no error" gaps.

**Architecture:** Two independent, same-shape fixes with zero shared code, so this plan is 2 tasks rather than 1. Task 1 extends `services/core/config`'s existing `Config.Validate()` with a `concurrency_limiter.default_max_concurrent` check, mirroring the already-shipped `fleet_shedder` check exactly — this gets fail-closed-at-startup and skip-bad-reload behavior for free via `main.go`'s existing `Validate()` call sites, no `main.go` changes needed. Task 2 extracts `services/sidecar/main.go`'s inline `RATECAP_MAX_INFLIGHT_REQUESTS` parsing into a small pure function so a range check (reject `<= 0`) can be unit-tested, since `main()` itself isn't testable (it calls `log.Fatalf` and blocks on `http.ListenAndServe`). A reviewer could approve one task while rejecting the other on its own merits — different modules, no interface dependency — so they are not merged into one task.

**Tech Stack:** Go 1.26 modules (`services/core`, `services/sidecar`), standard `go test` (with `-race`).

```go
	return nil
}
```

with:

```go
func (c *Config) Validate() error {
	if c.Tiers.ConcurrencyLimiter.DefaultMaxConcurrent <= 0 {
		return fmt.Errorf("tiers.concurrency_limiter.default_max_concurrent must be > 0, got %d (is the concurrency_limiter block missing from your config?)", c.Tiers.ConcurrencyLimiter.DefaultMaxConcurrent)
	}
	if c.Tiers.FleetShedder.DefaultMaxConcurrent <= 0 {
		return fmt.Errorf("tiers.fleet_shedder.default_max_concurrent must be > 0, got %d (is the fleet_shedder block missing from your config?)", c.Tiers.FleetShedder.DefaultMaxConcurrent)
	}
```

```go
func resolveMaxInflight(envVal string, defaultVal int64) int64 {
	if envVal == "" {
		return defaultVal
	}
	parsed, err := strconv.ParseInt(envVal, 10, 64)
	if err != nil {
		log.Printf("RATECAP_MAX_INFLIGHT_REQUESTS=%q is not a valid integer, using default of %d: %v", envVal, defaultVal, err)
		return defaultVal
	}
	if parsed <= 0 {
		log.Printf("RATECAP_MAX_INFLIGHT_REQUESTS=%q must be a positive integer, using default of %d", envVal, defaultVal)
		return defaultVal
	}
	return parsed
}
```

The rest of `main()` (dialing core, wiring `proxy.NewHandler`, starting the HTTP server) is unchanged.
